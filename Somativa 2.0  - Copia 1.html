# Processamento da DateTime APS a diviso 
def process_datetime(df):
    dt = pd.to_datetime(df['DateTime'])
    return pd.DataFrame({
        'Hour': dt.dt.hour,
        'DayOfWeek': dt.dt.dayofweek,
        'Month': dt.dt.month,
        'IsWeekend': dt.dt.dayofweek.isin([5, 6]).astype(int),
        'IsRushHour': dt.dt.hour.isin([7, 8, 17, 18, 19]).astype(int)
    }, index=df.index)

# Pr-processador
preprocessor = ColumnTransformer(
    transformers=[
        ('num', StandardScaler(), numeric_features),
        ('cat', OneHotEncoder(handle_unknown='ignore'), categorical_features)
    ])
In[9]:
# Pipeline
pipeline = Pipeline([
    ('preprocessor', preprocessor),
    ('feature_selector', SelectFromModel(XGBRegressor(random_state=42))),
    ('model', XGBRegressor(
        random_state=42,
        n_estimators=200,
        max_depth=6,
        learning_rate=0.1,
        subsample=0.8,
        colsample_bytree=0.8
    ))
])

Concluso da Equipe sobre as Mtricas

Com base nos resultados obtidos, a equipe conclui que os valores das mtricas de avaliao indicam um desempenho satisfatrio do modelo, embora haja espao para melhorias como: Aprimorar o pr-processamento, testar outros algoritmos de regresso, ajustar hiperparmetros.

O MAE de 214,67 demonstra que o erro mdio nas previses razovel, considerando a natureza do problema. Esse valor pode ser considerado bom, desde que esteja dentro de um intervalo tolervel para o domnio da aplicao (ex: se o total previsto for da ordem de milhares).

O RMSE de 344,46, por sua vez, sendo significativamente maior que o MAE, sugere que existem algumas previses com erros maiores possivelmente causadas por outliers ou variveis no explicadas adequadamente.

Em resumo, os resultados so bons para uma primeira verso, mas com intervenes pontuais, possvel alcanar uma performance ainda melhor.
